# @better-giving/paypal-sdk

Internal package (`private`), a **built** TypeScript SDK for PayPal REST APIs, generated from official OpenAPI specs. Tree-shakeable factory functions over generated per-API clients. Consumed by platform via `workspace:*` (not yet wired — platform still uses the separate `@better-giving/paypal`). The **only** member that emits + ships `dist/`.

## Layout

- **`specs/openapi/*.json`** — 13 committed vendored OpenAPI specs (one per PayPal API: orders, payments, invoicing, subscriptions, vault, payouts, partner-referrals, disputes, catalog, tracking, web-profiles, webhooks, transactions). The codegen input; the only vendored data that ships in git.
- **`scripts/generate-sdk.js`** — reads `specs/openapi`, runs `openapi-typescript-codegen` per spec → `generated/<api>/`. **`scripts/fix-generated.js`** — post-codegen fixups on `generated/`.
- **`generated/`** — codegen output. **Gitignored, build artifact — never edit, never commit.** Regenerated by `generate`.
- **`src/`** — hand-written: `client.ts` (`PayPalClient`, auth/env), `factories/*.ts` (13 `create_<api>_service` factories wrapping generated clients), `index.ts` (barrel).
- **`index.ts`** (root) — the emitted entry barrel; re-exports client + factories from `src/`.
- **`dist/`** — `tsc` output (`index.js`/`.d.ts` + `generated/`). **Gitignored, build artifact.** The `exports`/`main`/`types` all point here.

## Build

`build` = `clean` (rm `generated dist`) → `generate` (specs → `generated/`) → `tsc` (→ `dist/`). Runs in `turbo run build`; `outputs` caches both `generated/**` and `dist/**`. `dependsOn: ["^build"]` — nothing upstream, but the member is built before any consumer.

- **do NOT extend `tsconfig.base.json`** — the base is `noEmit`/`module:preserve` (transpile-by-platform policy). This package's `tsconfig.json` is **standalone** and emits (`declaration`, `outDir: dist`, `module: ESNext`, `moduleResolution: node`). Keep it that way; folding it into the base breaks the build.
- imports use explicit `.js` extensions (emitted ESM) — `./client.js`, `./factories/orders.js`. Match this; do not drop extensions.
- factory naming is `snake_case` (`create_orders_service`) — an SDK convention, kept as-is. Don't rename to match platform's camelCase.

## Conventions

- **No pre-commit `type-check`** (see root CLAUDE.md): `src/` imports gitignored `generated/` code that's absent pre-commit, so tsc can't run staged. Biome check covers staged files; type safety is enforced by the build in turbo/CI. After changing `src/` or a spec, run `pnpm --filter @better-giving/paypal-sdk build` yourself to type-check.
- **Biome** governs the whole member, incl. the root `index.ts` barrel (root `pnpm lint`/`format` run one `biome check .` over the repo; this member's own `lint`/`format` scripts scope to `.`); `generated/`, `dist/`, `specs/` are excluded in root `biome.json`.
- pin deps exact (repo-wide rule). `@openapitools/openapi-generator-cli` has a postinstall → listed in root `onlyBuiltDependencies`.
- adding a PayPal API: drop its spec in `specs/openapi/`, add a `factories/<api>.ts`, export from `src/index.ts` + root `index.ts`, add a subpath to `exports`, then `build`.
- **publishing is stripped** — private member, no changesets/release infra. If it ever needs to publish standalone again, restore `publishConfig`/changesets from git history.
